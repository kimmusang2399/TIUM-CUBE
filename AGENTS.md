# TIUM CUBE 개발 에이전트 지침

이 문서는 TIUM CUBE 홈페이지를 개발할 때 모든 작업자가 공통으로 따라야 할 기준입니다.

## 프로젝트 요약

- 프로젝트명: TIUM CUBE - 미래 식량 큐브 스마트팜
- 목적: 사업 홍보, 고객 유입, 전화 및 카카오톡 상담 전환
- 핵심 가치: 낮은 초기 투자비, 낮은 유지보수 비용, 쉬운 운영, AI 기반 스마트팜
- 구현 형태: 반응형 랜딩페이지

## 개발 원칙

- 첫 화면에서 TIUM CUBE가 어떤 사업인지 즉시 알 수 있어야 한다.
- 모든 섹션은 상담 전환으로 이어지는 흐름을 가져야 한다.
- 로그인, 회원가입, DB, 관리자, 결제, 게시판 기능은 구현하지 않는다.
- 정적 HTML, CSS, JavaScript 범위에서 완성한다.
- 전화 문의와 카카오톡 상담 링크는 실제 연결 가능한 값으로 교체하기 쉽게 관리한다.

## 우선순위

1. 모바일에서 상담 전환이 쉬운 구조
2. 기존 스마트팜 대비 비용 절감 메시지
3. 신뢰감 있는 디자인과 명확한 정보 구조
4. 빠른 로딩과 단순한 유지보수
5. 검색 노출과 공유 시 보기 좋은 기본 메타 정보

## 주요 사용자

- 청년 창업가
- 소상공인
- 농업인
- 기업 담당자
- 지자체 담당자

## 개발 시 주의사항

- 불필요한 기능을 추가하지 않는다.
- 과도한 애니메이션으로 정보 전달을 방해하지 않는다.
- 카드형 레이아웃은 사용하되, 화면이 조각나 보이지 않게 섹션별 흐름을 유지한다.
- 모바일 CTA는 항상 쉽게 누를 수 있는 위치와 크기를 유지한다.
- 상담 버튼 문구는 구체적인 행동을 유도해야 한다.
