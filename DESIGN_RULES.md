# TIUM CUBE 디자인 규칙

## 디자인 키워드

- 미래지향적
- 친환경
- 신뢰감
- 전문성
- 단순하고 명확한 정보 구조
- 고급스러운 B2B 홍보 랜딩페이지
- 제품 실물이 크게 보이는 첫 화면

## 폰트

- 1순위: Pretendard
- 2순위: Noto Sans KR
- 기본 대체: sans-serif

## 레이아웃

- 전체 구조는 랜딩페이지 단일 흐름으로 구성한다.
- 섹션 순서는 Header, Hero, 문제 제기, 비교, 기술, 효과, 적용분야, 절차, 상담, Footer 순서를 따른다.
- 정보는 한 번에 이해 가능한 단위로 나눈다.
- 카드형 레이아웃은 핵심기술, 도입효과, 적용분야에 사용한다.
- 비교표는 데스크톱에서는 표 형태, 모바일에서는 카드 형태로 읽히게 한다.
- Header는 흰색 배경, 좌측 브랜드, 중앙 메뉴, 우측 상담 버튼 구조를 기본으로 한다.
- Hero는 좌측 강한 메시지와 CTA, 우측 제품 이미지를 크게 배치한다.
- 섹션 제목은 중앙 정렬을 기본으로 하며 핵심 단어만 녹색으로 강조한다.
- 하단 상담 영역은 넓은 녹색 배너로 처리하고 제품 이미지를 함께 사용한다.

## 버튼

- CTA 버튼은 명확한 행동 문구를 사용한다.
- 주요 버튼: 무료 상담 신청
- 보조 버튼: 카카오톡 상담하기, 전화 문의하기
- 모바일에서는 손가락으로 누르기 쉬운 높이와 간격을 확보한다.
- 버튼 색상은 메인 색상 또는 강조 색상을 사용한다.
- 카카오톡 상담 버튼은 녹색 또는 노란색을 사용해 일반 버튼과 구분한다.
- 전화 문의 버튼은 흰색 배경과 녹색 테두리를 사용해 보조 CTA로 표현한다.

## 이미지와 아이콘

- 실제 제품, 큐브형 스마트팜, 작물, 실내 재배 이미지를 우선 사용한다.
- 임시 이미지를 사용할 경우 교체가 쉽도록 파일명과 위치를 명확히 한다.
- 아이콘은 기술, 비용 절감, 유지보수, 상담, 절차 설명에 제한적으로 사용한다.
- 모든 페이지 내 이미지는 `assets/images/` 안에 저장해 참조한다.
- 참고 디자인 원본은 `assets/images/reference-design.png`로 보관한다.
- Hero 제품 이미지는 `assets/images/hero-smartfarm.png`를 사용한다.
- 작물 이미지는 적용분야 카드에서 실제 썸네일로 사용한다.

## 애니메이션

- 스크롤 진입 애니메이션은 가볍게 사용한다.
- 정보 전달보다 애니메이션이 더 눈에 띄지 않게 한다.
- Hover 효과는 버튼과 카드에만 제한적으로 적용한다.

## 참고 이미지 기반 고도화 규칙

- 전체 배경은 흰색과 아주 연한 회색 계열을 중심으로 유지한다.
- 카드에는 얇은 테두리와 약한 그림자만 사용한다.
- 한 섹션 안에서는 아이콘, 제목, 설명의 반복 리듬을 맞춘다.
- 비교 영역은 좌측 기존 방식, 우측 큐브 스마트팜 구조로 명확히 대비한다.
- 도입효과는 숫자 성과를 짧고 굵게 보여준다.
- Footer는 짙은 녹색 계열로 마감해 신뢰감과 완성도를 준다.
